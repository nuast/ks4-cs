## What is this?

This is a free, open-source revision and practice scaffold organised **exactly** in the official OCR GCSE Computer Science (J277) specification order.

Every file maps directly to a specification section number so you always know where you are and what you need to know.

---

## How to use this resource

1. **Find your topic** using the [Contents / Specification Index](CONTENTS.md) or the component folders below.
2. **Open the topic file** – each one follows the same structure so navigation is always familiar.
3. **Work through each section** in order: summary → specification points → key terms → notes → examples → common mistakes → exam focus → quick check → exam-style practice → stretch.
4. **Use the links** at the bottom of each file to jump to related topics.
5. **Tick off specification points** as you feel confident with them.

## OCR command words to know

OCR uses specific command words in questions. Knowing what each one means helps you write the right kind of answer.

| Command word | What to do |
|---|---|
| **Define** | Give the precise meaning of a term |
| **Describe** | Give the key features; say what something is or does |
| **Explain** | Give reasons or say how/why something works |
| **Compare** | Give similarities **and** differences |
| **Calculate** | Show working and give a numerical answer |
| **Discuss** | Give different viewpoints; weigh up arguments |
| **Refine** | Improve an algorithm or program |
| **Trace** | Complete a trace table step by step |
| **Evaluate** | Weigh up strengths and weaknesses; reach a conclusion |
