# 1.5 Systems Software

> **Component:** J277/01 – Computer Systems | **Section:** 1.5 | **Exam:** J277/01

---

## What you need to know

Systems software is the software that manages the hardware and provides a platform for applications to run on. The most important piece of systems software is the **operating system (OS)**. You also need to know about **utility software** – programs that maintain and optimise the system.

## Key terms

| Term | Definition |
|------|-----------|
| Operating system (OS) | Systems software that manages hardware and provides an interface for users and applications |
| Memory management | The OS allocating RAM to programs that need it and reclaiming it when finished |
| Process management | The OS scheduling and coordinating programs that run on the CPU |
| Multi-tasking | The OS switching rapidly between processes so several appear to run simultaneously |
| Peripheral management | The OS communicating with input/output devices using drivers |
| Driver | Software that allows the OS to communicate with a specific hardware device |
| File management | The OS organising files and folders, including read/write access control |
| User interface (UI) | The way the user interacts with the system (GUI or CLI) |
| GUI | Graphical User Interface – windows, icons, menus, pointer |
| CLI | Command Line Interface – text-based commands entered by the user |
| Utility software | Programs that perform maintenance tasks to keep the system running smoothly |
| Defragmentation | Rearranging fragmented data on a magnetic hard drive to improve read speed |
| Encryption utility | Software that encrypts files or drives to protect data |
| Backup utility | Software that automatically copies data to a secondary location for recovery purposes |

## Core notes

### Role of the operating system

The OS sits between the hardware and the applications. Its main roles are:

**1. Memory management**
- Allocates sections of RAM to each running program.
- When a program closes, the memory is freed.
- Uses virtual memory when RAM is full.

**2. Process management (multitasking)**
- Manages which programs get CPU time and when.
- Uses **scheduling** algorithms to share CPU time fairly.
- Appears to run multiple programs simultaneously (rapid switching on a single core).

**3. Peripheral management**
- Uses **drivers** to communicate with hardware (printer, keyboard, monitor, etc.).
- Acts as an intermediary so programs don't need to know the technical details of each device.

**4. User interface**
- Provides a way for users to interact with the computer.
- **GUI** – intuitive, icons and windows, uses a pointing device; suitable for general users.
- **CLI** – text commands; requires knowledge of command syntax; faster for experienced users; less RAM needed.

**5. File management**
- Creates, organises, and manages the file system (folders, files, permissions).
- Controls who can read, write, or execute each file.
- Manages file paths, extensions, and metadata.

---

### Utility software

| Utility | Purpose |
|---------|---------|
| **Compression** | Reduces file size for storage or transfer |
| **Defragmentation** | Reorganises fragmented files on a magnetic HDD so they are stored contiguously (improves speed). Not needed for SSDs. |
| **Anti-malware / antivirus** | Detects, quarantines, and removes malicious software |
| **Backup** | Copies data to a secondary location so it can be restored after loss or corruption |
| **Disk formatter** | Prepares a storage device by creating a file system; erases all data |
| **Screen reader** | Reads on-screen text aloud to assist users with visual impairments (accessibility utility) |
| **Firewall** | Monitors network traffic to block unauthorised access |

---

### Fully-featured OS vs embedded OS

| | Fully-featured OS | Embedded OS |
|--|------------------|------------|
| Purpose | General-purpose (runs many applications) | Single dedicated purpose |
| Examples | Windows, macOS, Linux, Android | OS in a microwave, washing machine, car ECU |
| User interface | GUI or CLI | Often no interface (no screen/keyboard) |
| Resources used | High (needs RAM, storage, processing power) | Very low |
| Updatable? | Yes, regularly | Rarely / not at all |
| Cost | Higher | Lower |

## Worked examples

**Example 1 – OS memory management in action**

1. User opens a web browser → OS allocates 500 MB of RAM.
2. User opens a word processor → OS allocates 200 MB of RAM.
3. RAM becomes full → OS moves least-used data to virtual memory (HDD).
4. User closes the web browser → OS frees the 500 MB back to available RAM.

---

**Example 2 – GUI vs CLI comparison**

| Task | GUI | CLI |
|------|-----|-----|
| Create a folder | Click → New Folder → type name | `mkdir foldername` |
| Delete a file | Right-click → Delete → confirm | `rm filename` |
| Rename a file | Slow (right-click → Rename) | Very fast (`mv old new`) |

CLI is faster for repetitive tasks and scripting but requires knowing commands.

## Common mistakes

- ❌ Saying the OS "runs programs" – more accurately, the OS manages the resources so programs can run.
- ❌ Confusing a driver with the OS – a driver is a specific program that allows the OS to communicate with one device; the OS manages drivers.
- ❌ Saying defragmentation helps SSDs – SSDs do not fragment in the same way; defragmenting an SSD can actually reduce its lifespan.
- ❌ Saying backup "prevents" data loss – backup allows data to be *recovered* after loss; it does not stop the loss from happening.
- ❌ Confusing utility software with application software – utility software maintains the system; application software is used to do tasks (word processing, gaming, etc.).

## Quick-check questions

1. State **three** roles of an operating system.
2. What is a driver and why is it needed?
3. Describe **one** difference between a GUI and a CLI.
4. What is the purpose of defragmentation software and why is it not needed for solid-state drives?
5. Give **two** examples of utility software and describe what each one does.

<details>
<summary>Answers</summary>

1. Any three from: memory management; process management / multitasking; peripheral management; user interface; file management.
2. A driver is software that allows the operating system to communicate with a specific hardware device. It is needed because the OS cannot know the technical details of every possible device.
3. A GUI uses icons, windows, and a pointing device, making it intuitive for general users; a CLI uses text commands, which is faster for experienced users but requires knowledge of command syntax.
4. Defragmentation reorganises files on a magnetic hard drive that have been split into fragments, so they are stored contiguously, improving read speed. SSDs do not use moving read heads, so fragmented storage does not cause the same performance penalty.
5. Any two with descriptions, e.g. Anti-malware — detects and removes malicious software; Backup utility — automatically copies data to a safe location so it can be restored if lost.

</details>
