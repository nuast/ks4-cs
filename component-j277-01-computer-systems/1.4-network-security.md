# 1.4 Network Security

> **Component:** J277/01 – Computer Systems | **Section:** 1.4 | **Exam:** J277/01

---

## What you need to know

Any networked system can be attacked. This topic covers the main types of threat that networks face and the methods used to protect against them. You need to be able to describe threats and prevention methods clearly, and explain how specific attacks work.

---

## Specification points

> Tick each point when you feel confident with it.

- [ ] Describe common network threats: malware, phishing, brute force attacks, SQL injection, denial-of-service (DoS), social engineering, man-in-the-middle attacks
- [ ] Describe prevention methods: firewalls, encryption, strong passwords/authentication, anti-malware software, whitelisting, blacklisting, penetration testing

## Key terms

| Term | Definition |
|------|-----------|
| Malware | Malicious software designed to damage, disrupt or gain unauthorised access to a system |
| Virus | Malware that attaches to files and spreads when files are shared |
| Ransomware | Malware that encrypts the victim's data and demands payment to decrypt it |
| Spyware | Malware that secretly collects information about a user |
| Phishing | Fraudulent messages (usually email) that trick users into revealing sensitive information |
| Brute force attack | Trying every possible password combination until the correct one is found |
| SQL injection | Inserting malicious SQL code into an input field to manipulate a database |
| DoS attack | Flooding a server with traffic to make it unavailable to legitimate users |
| DDoS | Distributed DoS – the attack comes from many machines at once (a botnet) |
| Social engineering | Manipulating people into revealing confidential information |
| Man-in-the-middle | An attacker secretly intercepts and potentially alters communication between two parties |
| Firewall | Hardware or software that monitors and controls network traffic based on security rules |
| Encryption | Converting data into an unreadable form so only authorised parties can decode it |
| Penetration testing | Authorised simulated attacks on a system to identify and fix vulnerabilities |
| Whitelisting | Allowing only approved programs/users/IP addresses to access a system |
| Blacklisting | Blocking known malicious programs/users/IP addresses |

## Core notes

### Threats to networks

**Malware**
- Umbrella term for any malicious software.
- Types: virus, worm, Trojan horse, ransomware, spyware, adware.
- Spreads via email attachments, infected downloads, USB drives, malicious websites.

**Phishing**
- Fake emails or websites mimic trusted organisations (banks, HMRC, etc.).
- Goal: trick the user into entering login credentials or personal/financial data.
- Spear phishing = targeted phishing at a specific person or organisation.

**Brute force attack**
- Automated software tries every possible password combination.
- Easy passwords are cracked quickly; long, complex passwords take much longer.
- Countermeasure: lockout after a number of failed attempts.

**SQL injection**
- Attacker enters SQL code into a form field (e.g. login box, search box).
- If inputs are not sanitised, the code runs against the database.
- Can expose, modify, or delete data.
- Example: entering `' OR '1'='1` into a login field may bypass authentication.

**Denial-of-Service (DoS) / DDoS**
- Overwhelms a server with requests so it cannot serve legitimate users.
- DDoS uses a network of compromised machines (botnet).
- Prevents access to websites or online services.

**Social engineering**
- Exploits human psychology rather than technical weaknesses.
- Examples: pretexting (inventing a scenario), tailgating, vishing (voice phishing), blagging.

**Man-in-the-middle (MitM)**
- Attacker positions themselves between two communicating parties.
- Can read or modify messages without either party knowing.
- Common on unsecured Wi-Fi networks.

---

### Prevention methods

| Prevention | How it works |
|-----------|-------------|
| **Firewall** | Monitors incoming and outgoing network traffic; blocks unauthorised access based on rules |
| **Encryption** | Data is scrambled; even if intercepted, it cannot be read without the decryption key |
| **Strong passwords / multi-factor authentication** | Long, complex, unique passwords are harder to brute-force; MFA adds an extra verification step |
| **Anti-malware software** | Detects and removes known malware; regular updates needed for new threats |
| **Whitelisting** | Only explicitly approved software/users/IPs are allowed — blocks everything else by default |
| **Blacklisting** | Known malicious software/IPs are blocked — everything else is allowed by default |
| **Penetration testing** | Ethical hackers simulate attacks to find vulnerabilities before real attackers do |
| **User education** | Training staff to recognise phishing, social engineering, and safe password practices |
| **HTTPS / TLS** | Encrypts web traffic between browser and server (prevents MitM on web connections) |

## Worked examples

**Example 1 – SQL injection**

A login form sends the query:
```sql
SELECT * FROM users WHERE username='[input]' AND password='[input]';
```

If an attacker enters `admin' --` as the username:
```sql
SELECT * FROM users WHERE username='admin' --' AND password='anything';
```
The `--` comments out the password check, granting access without a valid password.

**Prevention:** validate and sanitise all input; use parameterised queries.

---

**Example 2 – Phishing email indicators**

| Red flag | Detail |
|---------|--------|
| Suspicious sender address | `support@paypa1.com` instead of `paypal.com` |
| Urgent language | "Act now or your account will be closed!" |
| Hover over links | URL goes to a fake site, not the real organisation |
| Generic greeting | "Dear Customer" instead of your name |
| Requests for credentials | Legitimate organisations never ask for passwords via email |

## Common mistakes

- ❌ Saying a firewall "removes viruses" – a firewall controls network traffic; it does not scan for or remove malware.
- ❌ Confusing **phishing** (deceptive messages) with **pharming** – pharming redirects users to fake websites by poisoning DNS.
- ❌ Saying whitelisting is the same as blacklisting – whitelisting blocks everything except what is approved; blacklisting blocks only what is known to be bad.
- ❌ Confusing DoS (one source) with DDoS (many sources / botnet).
- ❌ Saying encryption "stops hackers getting your data" – encryption means that if data is intercepted, it cannot be read without the key.

## Exam focus

- **Define** – malware, phishing, brute force, SQL injection, DoS/DDoS, social engineering, man-in-the-middle, firewall, encryption.
- **Describe** – how a specific attack works; how a specific prevention method works.
- **Explain** – why a particular prevention method is appropriate for a given threat.
- **Compare** – whitelisting vs blacklisting; DoS vs DDoS.
- **Evaluate** – which combination of security measures would best protect a given system (often worth 4–6 marks).

> OCR often gives a scenario (e.g. a school network) and asks you to identify threats and recommend prevention methods. Always match the prevention to the threat.

## Quick-check questions

1. What is a brute force attack?
2. Give **two** signs that an email might be a phishing attempt.
3. What is the difference between a DoS attack and a DDoS attack?
4. Describe how a firewall protects a network.
5. Why is penetration testing useful for an organisation?

<details>
<summary>Answers</summary>

1. A brute force attack involves automated software trying every possible password combination until the correct one is found.
2. Any two: suspicious/misspelled sender address; urgent language / threat of consequences; generic greeting; links going to unexpected URLs; requests for personal information or passwords.
3. A DoS attack comes from a single source; a DDoS (Distributed DoS) attack uses many machines (a botnet) simultaneously, making it harder to block.
4. A firewall monitors incoming and outgoing network traffic and blocks connections that do not meet the defined security rules / blocks unauthorised access attempts.
5. Penetration testing identifies security vulnerabilities before real attackers can exploit them, allowing the organisation to fix weaknesses and improve security.

</details>

*[Mark scheme: User education / training staff to recognise phishing emails (1), so staff are less likely to click malicious links or reveal credentials (1); email filtering software to detect and block suspicious emails (1), preventing phishing emails from reaching staff inboxes (1). ---

**Q4.** Discuss the use of whitelisting and blacklisting as network security measures. Include the advantages and disadvantages of each. [6 marks]

*[Mark scheme: Whitelisting — only approved software/users are allowed (1); very effective as all unknown software is blocked by default (1); disadvantage: time-consuming to maintain the approved list / may block legitimate software (1). Blacklisting — known malicious software/IPs are blocked (1); easier to implement as unknown software is allowed by default (1); disadvantage: new/unknown malware is not blocked until it is added to the blacklist (1). Award up to 6 marks total.]*
