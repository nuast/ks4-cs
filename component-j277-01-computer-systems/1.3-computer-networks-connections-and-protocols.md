# 1.3 Computer Networks, Connections and Protocols

> **Component:** J277/01 – Computer Systems | **Section:** 1.3 | **Exam:** J277/01

---

## What you need to know

A network lets computers share data and resources. This topic covers the different types of network, the hardware needed, how data travels (topologies and protocols), and the addressing systems that make it all work. You need to be able to compare network types, describe topologies, and explain what key protocols do.

## Key terms

| Term | Definition |
|------|-----------|
| LAN | Local Area Network – network covering a small geographical area (e.g. a school) |
| WAN | Wide Area Network – network spanning large distances, often using leased lines (e.g. the internet) |
| Router | Directs packets of data between networks; connects a LAN to the internet |
| Switch | Connects devices within a LAN; directs data only to the intended device |
| WAP | Wireless Access Point – allows wireless devices to connect to a wired network |
| NIC | Network Interface Card – hardware in a device that enables it to connect to a network |
| Protocol | An agreed set of rules for communication between devices |
| IP address | A unique numerical label assigned to each device on a network |
| MAC address | A unique physical hardware address burned into a NIC at manufacture |
| Packet | A small chunk of data with header information (source/destination address, sequence number) |
| Packet switching | Breaking data into packets and sending each independently across the network |
| DNS | Domain Name System – translates domain names (e.g. www.example.com) into IP addresses |
| Bandwidth | The maximum amount of data that can be transmitted per second (bits/s) |
| Latency | The delay between sending and receiving data |

## Core notes

### Network types: LAN vs WAN

| | LAN | WAN |
|--|-----|-----|
| Area | Small (building, campus) | Large (city, country, global) |
| Ownership | Usually one organisation | Often uses leased infrastructure (BT, ISPs) |
| Speed | Generally faster | Generally slower |
| Examples | School network, home network | Internet, bank network across branches |

### Factors affecting network performance
- **Bandwidth** – higher bandwidth means more data can flow per second.
- **Latency** – delays caused by distance or congestion slow data transfer.
- **Number of users** – more simultaneous users = less bandwidth each.
- **Type of transmission medium** – fibre optic faster than copper wire; wired faster than wireless.
- **Network hardware** – quality and speed of routers, switches.

### Client-server vs peer-to-peer

| | Client-server | Peer-to-peer |
|--|---------------|-------------|
| Structure | Central server(s) + client devices | All devices are equal |
| Management | Centralised – easier to manage and secure | Decentralised – harder to manage |
| Cost | Higher (server hardware needed) | Lower (no dedicated server) |
| Performance | Server can become bottleneck | Performance shared across devices |
| Use case | School networks, websites | Small home networks, file sharing |

### Networking hardware

| Hardware | Role |
|----------|------|
| **Router** | Directs data packets between networks; connects LAN to WAN/internet; uses IP addresses |
| **Switch** | Connects devices within a LAN; uses MAC addresses to send data to the correct device only |
| **WAP** | Creates a wireless local network; connects wirelessly to devices; connects to switch/router via cable |
| **NIC** | Enables a device to connect to a network (wired or wireless); has a unique MAC address |

### Transmission media

| | Wired (copper/fibre) | Wireless |
|--|---------------------|---------|
| Speed | Higher | Lower |
| Security | More secure (harder to intercept) | Less secure (signals can be intercepted) |
| Reliability | More reliable | Subject to interference |
| Flexibility | Less flexible (cables restrict movement) | More flexible (mobile devices) |
| Cost | Higher installation cost | Lower cost to add devices |

### Network topologies

**Star topology**
- All devices connect to a central switch/hub.
- ✅ One cable failure only affects that device; easy to add/remove devices; good performance.
- ❌ If the central switch fails, the whole network fails; more cable needed than bus.

**Bus topology**
- All devices share one backbone cable.
- ✅ Simple and cheap to install; less cable than star.
- ❌ If the backbone fails, the entire network fails; performance degrades with more devices; data collisions.

**Mesh topology**
- Every device connects to every other device (full mesh) or to multiple devices (partial mesh).
- ✅ Highly fault-tolerant; no single point of failure; multiple paths for data.
- ❌ Expensive and complex to install and maintain; lots of cabling.

**Ring topology**
- Devices connect in a circle; data travels in one direction.
- ✅ Equal access for all devices; no data collisions.
- ❌ One failure can affect the whole network; harder to add/remove devices.

### Protocols and the TCP/IP model

A **protocol** is a set of rules that controls how data is formatted, sent, and received.

The **TCP/IP model** has 4 layers:

| Layer | Name | Examples |
|-------|------|---------|
| 4 | Application | HTTP, HTTPS, FTP, SMTP, IMAP, POP, DNS |
| 3 | Transport | TCP, UDP |
| 2 | Internet | IP |
| 1 | Network Access / Link | Ethernet, Wi-Fi |

### Common protocols

| Protocol | Full name | Use |
|----------|-----------|-----|
| HTTP | HyperText Transfer Protocol | Transfers web pages (unencrypted) |
| HTTPS | HTTP Secure | Encrypted web browsing |
| FTP | File Transfer Protocol | Transferring files between computers |
| SMTP | Simple Mail Transfer Protocol | Sending emails |
| IMAP | Internet Message Access Protocol | Receiving/managing emails on the server |
| POP | Post Office Protocol | Downloading emails to a local device |
| TCP | Transmission Control Protocol | Reliable, ordered delivery; checks for errors |
| UDP | User Datagram Protocol | Fast, unreliable delivery (used for streaming/gaming) |
| IP | Internet Protocol | Addressing and routing packets across networks |
| DNS | Domain Name System | Translates domain names to IP addresses |

### IP addresses and MAC addresses

| | IP address | MAC address |
|--|------------|-------------|
| What is it? | Logical address assigned to a device on a network | Physical address burned into the NIC |
| Can it change? | Yes (can be dynamic or static) | No (permanent) |
| Used by | Routers (to route packets between networks) | Switches (to deliver data within a LAN) |
| Format | 32-bit (IPv4, e.g. 192.168.1.1) or 128-bit (IPv6) | 48-bit hexadecimal (e.g. 00:1A:2B:3C:4D:5E) |

### Circuit switching vs packet switching

| | Circuit switching | Packet switching |
|--|------------------|-----------------|
| How? | A dedicated path is established for the whole communication | Data split into packets; each takes any available route |
| Efficiency | Wastes bandwidth if path idle | Efficient; bandwidth shared |
| Reliability | Fails if path breaks | Packets rerouted automatically |
| Use | Traditional telephone calls | Internet, most modern networks |

## Worked examples

**Example 1 – DNS in action**

You type `www.example.com` in your browser:

1. Browser asks the **DNS server** to look up `www.example.com`.
2. DNS returns the IP address (e.g. `93.184.216.34`).
3. Browser sends an HTTP/HTTPS request to that IP address.
4. Web server responds with the web page.

---

**Example 2 – Packet switching**

A 1,500-byte file is broken into three 500-byte packets:
- Packet 1 travels via Router A → Router C.
- Packet 2 travels via Router B → Router C (different route due to congestion).
- Packet 3 travels via Router A → Router C.
- Packets arrive, are reassembled in the correct order using sequence numbers, and the file is reconstructed.

## Common mistakes

- ❌ Confusing **router** (connects networks; uses IP addresses) with **switch** (connects devices within a LAN; uses MAC addresses).
- ❌ Saying WAP is the same as a router – a WAP provides wireless access to the LAN; it doesn't route data between networks.
- ❌ Confusing **bandwidth** (capacity) with **speed** – bandwidth is the maximum data rate; actual speed depends on many factors.
- ❌ Saying TCP is always better than UDP – UDP is better for time-sensitive applications (streaming, gaming) where a little data loss is acceptable.
- ❌ Thinking a star topology has no single point of failure – the central switch IS a single point of failure.

## Exam focus

- **Define** – LAN, WAN, protocol, router, switch, IP address, MAC address, DNS, packet switching.
- **Describe** – the role of a specific protocol (what it is used for).
- **Compare** – client-server vs peer-to-peer; star vs bus topology; TCP vs UDP; wired vs wireless.
- **Explain** – what happens when you type a web address (involves DNS); how packet switching works.
- **Identify** – which protocol is used for a given scenario.

> OCR often gives a scenario (e.g. "a school wants to share files securely") and asks which network type, topology, or protocol is most suitable — always justify your choice.

## Quick-check questions

1. What is the difference between a LAN and a WAN?
2. State the role of a DNS server.
3. Give **two** advantages of a star topology over a bus topology.
4. What is the difference between an IP address and a MAC address?
5. Give a situation where UDP would be preferred over TCP and explain why.

<details>
<summary>Answers</summary>

1. A LAN covers a small geographical area (e.g. a building) and is usually owned by one organisation; a WAN covers a large area (e.g. a country or globally) and often uses leased infrastructure.
2. A DNS server translates domain names (e.g. www.bbc.co.uk) into IP addresses so that data can be routed to the correct server.
3. Any two: if one cable fails, only that device is affected (not the whole network); easy to add or remove devices; better performance as less data collision risk.
4. An IP address is a logical address that can change and is used by routers to route packets between networks; a MAC address is a permanent physical address built into the NIC and used by switches within a LAN.
5. Live video streaming / online gaming — UDP is preferred because speed is more important than perfect reliability; a small amount of lost data is acceptable and requesting retransmission would cause delays.

</details>
