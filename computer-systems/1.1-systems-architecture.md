# 1.1 Systems Architecture

> **Component:** J277/01 – Computer Systems | **Section:** 1.1 | **Exam:** J277/01

---

## What you need to know

The CPU (Central Processing Unit) is the brain of the computer. This topic covers how the CPU is structured, how it processes instructions, and what affects its performance. You also need to know what embedded systems are and how they differ from general-purpose computers.

## Key terms

| Term | Definition |
|------|-----------|
| CPU | Central Processing Unit – the component that processes instructions |
| ALU | Arithmetic Logic Unit – performs calculations and logical comparisons |
| CU | Control Unit – directs the operation of the CPU; fetches, decodes and executes instructions |
| MAR | Memory Address Register – holds the address of the memory location being accessed |
| MDR | Memory Data Register – holds the data fetched from or to be written to memory |
| PC | Program Counter – holds the address of the next instruction to be fetched |
| Accumulator | Register that holds the result of the most recent ALU operation |
| Clock speed | Number of fetch-decode-execute cycles per second, measured in GHz |
| Cache | Small, fast memory inside the CPU that stores recently/frequently used data |
| Core | An independent processing unit within a CPU; more cores can handle more tasks simultaneously |
| Embedded system | A dedicated computer built into a device for a specific purpose (e.g. washing machine, microwave) |
| Von Neumann architecture | The stored-program design where data and instructions share the same memory and bus |

## Core notes

### The purpose of the CPU
The CPU carries out the instructions of a program. It fetches each instruction from memory, decodes what it means, and executes it.

### Von Neumann architecture
- Data and instructions are stored together in RAM.
- The CPU fetches instructions from memory along a shared bus.
- Key idea: programs are stored in memory, not hard-wired.

### CPU components

| Component | Role |
|-----------|------|
| ALU | Does maths (add, subtract, multiply) and logic (AND, OR, NOT, comparisons) |
| CU | Directs all CPU activity; controls the fetch-decode-execute cycle |
| PC | Keeps track of where the next instruction is in memory |
| MAR | Holds the address of the memory location the CPU wants to read or write |
| MDR | Holds the actual data being transferred to/from memory |
| Accumulator | Temporarily stores results from the ALU |

**Buses:**
- **Address bus** – carries the memory address (one direction only)
- **Data bus** – carries data between CPU and memory (both directions)
- **Control bus** – carries control signals (read/write, clock, interrupts)

### The fetch-decode-execute cycle

1. **Fetch** – the PC holds the address of the next instruction. The MAR copies the address. The instruction is fetched from RAM into the MDR. The PC increments (adds 1) ready for the next instruction.
2. **Decode** – the CU decodes the instruction in the MDR.
3. **Execute** – the ALU or another component carries out the instruction.

The cycle repeats continuously while the computer is running.

### Factors affecting CPU performance

| Factor | How it affects performance |
|--------|---------------------------|
| **Clock speed** | More cycles per second = faster processing |
| **Number of cores** | More cores = more instructions handled at once (multi-core processing) |
| **Cache size** | Larger cache = more data available near the CPU quickly = fewer slow RAM accesses |

> Note: doubling the clock speed does not always double overall performance because other bottlenecks exist (e.g. RAM speed, cache misses).

### Embedded systems
- A computer designed for **one specific task**, built into a larger device.
- Examples: washing machine controller, car's ABS braking system, digital thermostat, traffic light controller.
- Usually has: limited memory, no keyboard/screen, real-time operating requirements.
- Contrast with a **general-purpose computer** (PC/laptop) which runs many different programs.

## Worked examples

**Example 1 – Tracing the FDE cycle**

A simple program has the instruction `ADD 5` stored at memory address 200.

| Step | Register | Value | Action |
|------|----------|-------|--------|
| Fetch | PC → MAR | 200 | Address of next instruction copied to MAR |
| Fetch | RAM → MDR | `ADD 5` | Instruction fetched from address 200 |
| Fetch | PC | 201 | PC incremented |
| Decode | CU | `ADD 5` | CU interprets the instruction |
| Execute | ALU | Acc + 5 | ALU adds 5 to the accumulator |

---

**Example 2 – CPU performance comparison**

| CPU | Clock speed | Cores | Cache |
|-----|-------------|-------|-------|
| A | 2.0 GHz | 2 | 4 MB |
| B | 2.0 GHz | 4 | 4 MB |
| C | 3.5 GHz | 2 | 8 MB |

- CPU B will handle multi-threaded tasks better than CPU A (more cores).
- CPU C will be faster for single-threaded tasks (higher clock speed, larger cache).

## Common mistakes

- ❌ Confusing the **MAR** (holds an address) with the **MDR** (holds data) – remember: MAR = Address, MDR = Data.
- ❌ Saying the PC "runs programs" – the PC only stores the *address* of the next instruction.
- ❌ Thinking more cores always means faster – cores help with parallel tasks but not all programs use multiple cores.
- ❌ Confusing cache with RAM – cache is *inside* the CPU and much faster but much smaller.
- ❌ Saying embedded systems "cannot run programs" – they do run a program, just one fixed/dedicated program.

## Quick-check questions

1. What does the ALU do?
2. What is held in the Program Counter?
3. List the three buses found in the Von Neumann architecture and state what each carries.
4. Give **two** factors that affect CPU performance and explain how each one does so.
5. Give **two** examples of embedded systems and explain why they are not general-purpose computers.

<details>
<summary>Answers</summary>

1. The ALU performs arithmetic operations (add, subtract, multiply, divide) and logical comparisons (AND, OR, NOT, greater than, less than, equal to).
2. The address of the next instruction to be fetched from memory.
3. Address bus – carries memory addresses; Data bus – carries data between CPU and memory; Control bus – carries control signals (read/write, clock, interrupts).
4. Any two from: **Clock speed** – more cycles per second so more instructions processed per second; **Number of cores** – multiple cores can process instructions simultaneously; **Cache size** – more frequently used data stored near the CPU so fewer slow RAM accesses needed.
5. Any two examples (e.g. washing machine controller, car braking system, digital thermostat). They are not general-purpose because they are designed to run only one fixed program/task, not multiple user-selected programs.
