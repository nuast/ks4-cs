## Exam tips for J277/02

- You are expected to **read, trace, write, and refine** algorithms and programs.
- OCR uses its own **Exam Reference Language (ERL)** in exam questions – make sure you know the ERL syntax.
- Practice completing **trace tables** step by step; show every variable change.
- When writing algorithms or code, use clear structure: sequence → selection → iteration.
- Logic questions (**2.4**) often appear as truth tables or logic diagrams – both could appear.
- **Decomposition** and **abstraction** questions appear in 2.1 – know what each term means and give an example.
- Testing questions ask you to identify correct **normal**, **boundary**, and **erroneous** test data.

## OCR Exam Reference Language quick reference

Full ERL is covered in [2.2 Programming fundamentals](2.2-programming-fundamentals.md), but here are the key patterns:

```
// Output
print("Hello")

// Input
name = input("Enter name: ")

// Selection
if condition then
    ...
else
    ...
endif

// Count-controlled iteration
for i = 1 to 10
    ...
next i

// Condition-controlled iteration
while condition
    ...
endwhile

// Procedure
procedure myProc(param)
    ...
endprocedure

// Function
function myFunc(param)
    return value
endfunction
```
